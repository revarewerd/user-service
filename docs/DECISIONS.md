# User Service — Решения (ADR)

## ADR-001: Doobie вместо Quill

**Контекст:** Нужен типобезопасный доступ к PostgreSQL.  
**Решение:** Doobie — совместим с ZIO через interop-cats, явные SQL запросы.  
**Следствие:** Полный контроль над SQL, лёгкая отладка.

## ADR-002: BCrypt для паролей

**Контекст:** Нужен надёжный хеш паролей.  
**Решение:** jBCrypt — стандарт индустрии, salt встроен.  
**Следствие:** Проверка пароля ~100ms, но безопасно.

## ADR-003: Redis кэш прав с TTL 1h

**Контекст:** Проверка прав на каждый запрос — нагрузка на БД.  
**Решение:** Кэшировать в Redis (TTL 1h), инвалидировать при смене роли.  
**Следствие:** Проверка < 5ms из кэша.

## ADR-004: Иерархия ролей по level

**Контекст:** Нельзя позволить менеджеру назначить admin.  
**Решение:** Числовой level: 0 (super_admin) → 50 (viewer). Назначать можно только роли с level >= свой.  
**Следствие:** Простая и надёжная проверка.
